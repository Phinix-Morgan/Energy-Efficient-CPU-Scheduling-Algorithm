Architecture

System components and data flow diagram

System Architecture Diagram
🔴 Real-Time Tasks
• Algorithm: EDF
• Deadline guaranteed
• Freq scaled for deadline
• Preempts all others
• Slack-based DVFS
🔵 Interactive Tasks
• Algorithm: Priority RR
• Soft deadline
• Energy-aware freq
• Aging prevents starvation
• Balanced DVFS
🟤 Background Tasks
• Algorithm: Deferred FIFO
• No deadline
• Minimum frequency
• Run only when idle
• Max energy savings

⏱ Complexity Analysis

🏆 Key Design Decisions
✅ Task Packing — saves 15–40% energy
✅ Cubic DVFS Law — E ∝ f² advantage
✅ Break-even C-States — avoids costly wakes
✅ Aging Promotion — zero starvation
✅ Thermal Throttle — 80% at >75°C
✅ Slack Analysis — minimum freq for RT
